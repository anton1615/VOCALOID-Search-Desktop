# VOCALOID Search Desktop

ニコニコ動画のVOCALOID動画を検索するデスクトップアプリケーション。

## リリースノート

### v1.5.0 (2025-03-12)

#### プレイリスト状態同期の修正

- **「断層」問題を修正**: 検索、履歴、後で見るを切り替えた後、さらに読み込むと51番目の動画で順序が崩れる問題を解決
- **再生状態の独立性**: ページを切り替えても再生中の動画は継続され、他のリストから新しい動画を選ぶまで変わりません
- **バージョン管理**: リストコンテキストのバージョン管理を追加し、並行読み込みリクエストによるデータ破損を防止
- **後方互換性**: スムーズなアップグレードのためにレガシーフィールドを保持

#### 技術的改善

- `ListContext` と `ActivePlayback` モデルで Rust 状態管理をリファクタリング
- フロントエンドの復元ロジックが `list_context` から状態を読み取り、データの一貫性を確保
- 並行 `load_more` リクエストを無効化するバージョン予約メカニズムを追加

## 機能

- 高度なフィルタ検索（再生数、マイリスト数、コメント数、いいね数、投稿日時）
- カスタム数式によるソートとフィルタリング
- ピクチャー・イン・ピクチャー（PiP）モード
- 視聴履歴の記録
- ダーク/ライトテーマ対応
- 多言語対応（繁体字中国語、日本語、英語）

## インストール

### ポータブル版（USBメモリ）

1. `vocaloid-search-desktop.exe` をダウンロード
2. 実行ファイルと同じ場所に `data` フォルダを作成
3. アプリを実行 - すべてのデータは `data` フォルダに保存されます

```
/your-usb-drive/
├── vocaloid-search-desktop.exe
└── data/
    ├── data.db
    └── config.json
```

### 通常版

1. 実行ファイルをダウンロードして実行
2. データは `%APPDATA%/vocaloid-search-desktop/` に保存されます

## 動作環境

- Windows 10 以降
- WebView2 Runtime（通常 Windows 10/11 にプリインストール済み）

WebView2 がインストールされていない場合、自動的にインストールが促されます。

## 使い方

### 初回起動

1. 空のデータベースを検出し、スクレイパーページに移動します
2. 検索クエリを設定（デフォルト：「VOCALOID」）
3. 「同期開始」をクリックしてニコニコから動画データをダウンロード
4. 同期完了までお待ちください（大容量データは数時間かかる場合があります）

### 検索

- 検索バーにキーワードを入力して Enter キー
- フィルターバーで再生数、マイリスト数、コメント数、いいね数、投稿日でソート
- 歯車アイコンで高度なフィルターを開く
- 「視聴済みを非表示」で既に見た動画を除外

### 再生

- 動画サムネイルをクリックして埋め込みプレイヤーで再生
- PiPボタンで別の常に手前に表示されるプレイヤーウィンドウを開く
- 自動再生を有効にすると次の動画を自動的に再生

### カスタム数式

複数の指標に基づいて重み付けスコアを作成：
- スコア = (再生数重み × 再生数) + (マイリスト重み × マイリスト数) + (コメント重み × コメント数) + (いいね重み × いいね数)

## データベースの鮮度

アプリは起動時にデータベースの鮮度をチェックします：
- ニコニコスナップショットAPIは毎日 JST 05:00 頃に更新
- データベースが今日 06:00 より前に更新されている場合、同期が促されます

## キーボードショートカット

- `Enter` - 検索
- `Space` - 再生/一時停止（プレイヤーフォーカス時）
- `←` / `→` - 前の動画/次の動画

## 技術仕様

- Tauri 2.x で構築（Rust バックエンド + Vue 3 フロントエンド）
- SQLite データベース（FTS5 全文検索対応）
- ニコニコスナップショットAPIを使用

## 開発

```bash
# 依存関係のインストール
npm install

# 開発モードで実行
npm run tauri dev

# リリースビルド
npm run tauri build -- --no-bundle
```
