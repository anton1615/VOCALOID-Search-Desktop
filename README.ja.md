# VOCALOID Search Desktop

ニコニコ動画のVOCALOID動画を検索できる、モダンなSpotify風UIのローカルファースト・デスクトップアプリケーション。

> **[ニコニコ超検索](https://gokulin.info/search/) に触発されました** – 人気のニコニコ動画検索サービス。

これは [ウェブ版VOCALOID Search](https://github.com/anton1615/VOCALOID-Search) をTauriでローカルファーストのデスクトップアプリとして移植したものです。

> **注**: このプロジェクトは完全に **vibe coding** で構築されました – AIアシストによる反復開発で作成されています。

## 機能

- **モダンなインターフェース**: Spotify風のUI、プレイリスト形式のレイアウト
- **ライト/ダークモード**: テーマ切替対応
- **多言語対応**: 英語、日本語、中国語（繁体字）
- **PiPウィンドウ**: 常に最前面に表示されるピクチャー・イン・ピクチャープレイヤー
- **視聴履歴**: 視聴した動画の記録
- **視聴済み除外**: 検索結果から既に見た動画を除外
- **ウィンドウ状態の保存**: ウィンドウ位置、サイズ、最大化状態をセッション間で記憶
- **ローカルデータベース**: 動画データをローカルに保存し、オフラインで高速検索
- **カスタム数式での並べ替え・フィルタリング**: 再生数、マイリスト数、コメント数、いいね数を独自の重み付けでソート
- **オートスキップ**: 動画のエンディングを自動スキップ（エンドロールやスポンサー部分のスキップに便利）
- **埋め込みプレイヤー**: 公式ニコニコ埋め込みプレイヤーによる連続再生
- **キーワード+タグ検索**: タグフィルタ付き全文検索
- **無限スクロール**: 固定ページネーションではなく動的読み込み

## ウェブ版とデスクトップ版の比較

| 機能 | ウェブ版 | デスクトップ版 |
|------|---------|---------------|
| **デプロイ** | セルフホストサーバー (NixOS/Linux) | ローカルアプリ (Windows) |
| **稼働** | 24時間サーバー運用 | オンデマンド起動 |
| **スクレイパー** | systemdタイマーで自動実行 | 手動実行 |
| **マルチユーザー** | 対応（ユーザー登録可能） | 単一ユーザー、ローカルデータ |
| **モバイル対応** | PWA/TWAでAndroid対応 | 非対応 |
| **PiPモード** | コンパクトウィンドウにリサイズ | ネイティブ常時最前面ウィンドウ |
| **データ保存** | サーバー側データベース | ローカルファイルシステム |
| **オフライン検索** | サーバー接続が必要 | 完全ローカル |
| **プラットフォーム** | NixOS/Linux限定 | Windows (Linux対応予定) |

### どちらを選ぶべき？

**ウェブ版が適している場合:**
- 24時間自動スクレイピングを希望
- マルチユーザー対応が必要
- PWAでモバイルアクセスを希望
- NixOS/Linuxサーバーを運用している

**デスクトップ版が適している場合:**
- ネイティブデスクトップ体験を希望
- OSと密結合したPiPウィンドウを希望
- サーバー管理をしたくない
- 完全オフライン検索が必要

## 技術仕様

| レイヤー | 技術 |
|---------|------|
| **フロントエンド** | TypeScript, Vue 3, Vite, Tailwind CSS |
| **バックエンド** | Rust (Tauri 2.x) |
| **データベース** | SQLite (FTS5 全文検索) |
| **データソース** | [ニコニコ Snapshot API v2](https://site.nicovideo.jp/search-api-docs/snapshot.html), [GetThumbInfo API](https://site.nicovideo.jp/search-api-docs/thumb-info.html) |

### 実装アーキテクチャ

```
┌─────────────────────────────────────────┐
│           Vue 3 Frontend                │
│  (Search UI, Player, Settings, PiP)     │
└─────────────────┬───────────────────────┘
                  │ Tauri IPC
┌─────────────────▼───────────────────────┐
│           Rust Backend                   │
│  (SQLite, HTTP Client, File System)     │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│    Local SQLite Database (FTS5)         │
└─────────────────────────────────────────┘
```

### 主な技術的詳細

- **カスタムプロトコル**: HTTP localhostの代わりにTauriのカスタムプロトコル（`tauri://localhost`）を使用。ニコニコ埋め込みプレイヤーのドメイン制限を回避
- **FTS5全文検索**: SQLite FTS5でAND/OR/NOT演算子を使った高速なキーワード・タグ検索を実現
- **数式ベースのスコアリング**: 独自の重み付けで柔軟な並べ替えが可能

## 動作環境

- **OS**: Windows 10/11 (x64)
- **メモリ**: 最低4GB、推奨8GB
- **ストレージ**: アプリ約500MB + データベース（サイズは変動）
- **ネットワーク**: 動画再生とデータ同期にインターネット接続が必要

## データ保存先

データベースと設定は以下の場所に保存されます：
```
Windows: %APPDATA%\com.vocaloid-search.desktop
```

### ポータブルモード

ポータブルモード（データをアプリケーションフォルダに保存）を使用するには、実行ファイルと同じディレクトリに`data/`フォルダを作成してください：
```
<vocaloid-search-desktop.exeの場所>/data/
```

ポータブルモードが有効な場合、すべてのデータ（データベース、設定、サムネイル）がこのフォルダに保存されます。フォルダごと別のコンピュータにコピーすれば、同じデータを使用できます。

**注意**: ポータブルモードと標準モードの切り替えでは、データは自動的に移行されません。

## 使い方

### 利用の流れ

1. **起動時チェック**: アプリ起動時、データベースが空か古いかを自動チェックします。ニコニコSnapshot APIは毎日JST 5-6時頃に更新されます。データベースが古い場合、同期を促すメッセージが表示されます。

2. **データ同期**: 「データ同期」ページでスクレイパーの設定と実行：
   - **クエリ**: 検索キーワード（デフォルト: `VOCALOID`）
   - **最大日数**: 過去N日以内の動画のみ取得（デフォルト: 365、空欄で無制限）
   - **検索対象**: `tags`、`tagsExact`、`title`、`description`、または`tags,title`のような組み合わせ
   - **カテゴリ**: カテゴリフィルタ（デフォルト: `MUSIC`）
   
   「同期開始」をクリックして動画データをダウンロード。1日1回の同期をおすすめします。

3. **検索・閲覧**: 検索バーでキーワード検索。タグフィルタで結果を絞り込み。ソート数式で再生数やマイリスト数などの重み付けを調整。「視聴済み除外」を有効にすると、既に見た動画を非表示にできます。

4. **動画視聴**: 動画をクリックすると埋め込みプレイヤーで再生。リストから連続自動再生。再生設定でオートスキップを有効にすると、動画のエンディングを自動スキップできます。

5. **PiPモード**: ポップアウトボタンでPiPウィンドウを開く。ナビゲーションのない簡易プレイヤーで、常に最前面に表示。

### スクレイパー設定オプション

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `query` | `VOCALOID` | ニコニコSnapshot APIの検索キーワード |
| `max_age_days` | `365` | 過去N日以内の動画のみ取得。空欄で無制限 |
| `targets` | `tags` | 検索対象: `tags`、`tagsExact`、`title`、`description`、または組み合わせ |
| `category_filter` | `MUSIC` | ニコニコカテゴリフィルタ（MUSIC, GAME, ANIME, ENTERTAINMENT, DANCE, OTHER） |

## ソースからのビルド

### 前提条件

- Node.js 18+
- Rust 1.70+ (`cargo`含む)
- Windows 10/11 SDK

### ビルド手順

```bash
# リポジトリをクローン
git clone https://github.com/anton1615/VOCALOID-Search-Desktop.git
cd VOCALOID-Search-Desktop/vocaloid-search-desktop

# 依存関係をインストール
npm install
```

#### 開発ビルド

```bash
npm run tauri dev
```

フロントエンド開発サーバーとTauriが開発モードで起動します。

#### 本番ビルド

```bash
# フロントエンドとバックエンドを一緒にビルド
npm run tauri build

# またはデバッグフラグ付きでビルド（バイナリは大きくなるが診断しやすい）
npm run tauri build -- --debug
```

> **重要**: Tauriは埋め込みプレイヤー用にHTTP localhostの代わりにカスタムプロトコル（`tauri://localhost`）を使用します。これはニコニコの埋め込みプレイヤーが`localhost`ドメインからのリクエストを拒否するためです。本番ビルドで正しく処理されますが、問題が発生した場合は`--debug`フラグを試してください。

ビルドされた実行ファイルの場所：
```
vocaloid-search-desktop/src-tauri/target/release/vocaloid-search-desktop.exe
```

## 既知の問題と今後の計画

> ⚠️ **プロトタイプ版**: 早期プロトタイプです。機能が不安定で、テストされていない部分があります。

### 既知の問題

1. **履歴ページ**: 視聴記録は表示されますが、クリックしても動作しません
2. **PiP ↔ メインウィンドウ同期**: 状態同期が不完全
   - PiPでの再生が履歴に記録されません
   - PiPを開く/閉じる際、再生状態がリセットされます
3. **PiPウィンドウ**: まれに閉じられないことがあります（原因不明）
4. **PiPプレイリスト読み込み**: PiPが読み込み済み結果の最後に達すると、メインウィンドウがさらに読み込むのを待ちます（PiP自体は追加読み込みをトリガーできません）
5. **地域制限動画**: 自動再生が中断されます。再生できないため視聴済みにも記録されません
6. **その他**: 多くのエッジケースが未テストです

### 今後の計画

1. **「あとで見る」機能**: ニコニコの「あとで見る」のような機能を実装予定
2. **カスタムプレイリスト**: ユーザー定義の動画コレクション
3. **状態同期の改善**: メインウィンドウとPiP間の同期改善
4. **Linux対応**: Tauriのクロスプラットフォーム機能を活用したネイティブLinuxビルド
